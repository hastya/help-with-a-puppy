# 📱 Android-приложение «Help with a puppy»

Мобильная версия — это **автономное приложение**: оно работает полностью
**офлайн**, все данные хранятся прямо на телефоне (никакого сервера не нужно).
Технически это тот же интерфейс, что и веб-версия, но слой доступа к данным
заменён на локальное хранилище (`localStorage`), а сборка упакована в APK с
помощью [Capacitor](https://capacitorjs.com/).

```
mobile/
├── www/                     # автономное веб-приложение (офлайн)
│   ├── index.html
│   ├── js/
│   │   ├── calc.js          # расчёты (порт с сервера)
│   │   ├── breeds.js        # база пород
│   │   ├── store.js         # хранилище на localStorage
│   │   ├── api-local.js     # локальная реализация всех API-маршрутов
│   │   └── ui/views/app…    # тот же UI, что и в вебе
│   ├── manifest.webmanifest # PWA-манифест
│   ├── service-worker.js    # офлайн-кэш оболочки
│   └── icons/               # иконки приложения
├── android/                 # нативный Android-проект (Capacitor)
├── capacitor.config.json
└── package.json
```

## 🚀 Как получить APK и установить на телефон

### Способ 1 — скачать готовый APK (проще всего)

APK собирается автоматически через **GitHub Actions** и публикуется в разделе
**Releases**:

1. Откройте страницу релизов:
   **<https://github.com/hastya/help-with-a-puppy/releases>**
2. В релизе **«Android APK (latest)»** скачайте файл
   **`help-with-a-puppy.apk`** прямо на телефон
   (постоянная ссылка: `.../releases/download/android-latest/help-with-a-puppy.apk`).
3. Откройте скачанный файл на телефоне. Android попросит **разрешить установку
   из этого источника** — подтвердите.
4. Готово: иконка 🐶 «Help with a puppy» появится на рабочем столе. Приложение
   работает без интернета.

> Если релиза ещё нет — запустите сборку вручную: вкладка **Actions → Build
> Android APK → Run workflow**. После завершения (~5–10 мин) APK появится и в
> артефактах сборки, и в Releases.

### Способ 2 — установить как PWA (без APK)

Веб-версию можно «установить» без APK: откройте сайт приложения в Chrome на
Android и выберите **«Добавить на главный экран»**. Появится иконка, приложение
откроется в полноэкранном режиме и будет работать офлайн.

## 🛠️ Локальная сборка APK (для разработчиков)

Требуется **Node.js ≥ 18**, **JDK 17+** и **Android SDK** (проще всего через
Android Studio).

```bash
cd mobile
npm install
npx cap sync android          # копирует www/ в нативный проект

# Вариант А — собрать APK из командной строки
cd android
./gradlew assembleDebug
# готовый файл: mobile/android/app/build/outputs/apk/debug/app-debug.apk

# Вариант Б — открыть в Android Studio
cd ..
npx cap open android          # затем Run ▶ на эмуляторе/телефоне
```

Установить APK на подключённый по USB телефон:

```bash
adb install mobile/android/app/build/outputs/apk/debug/app-debug.apk
```

## 🔧 Как это устроено

- **Тот же UI.** Файлы `ui.js`, `views.js`, `views-sections.js`, `app.js`
  идентичны веб-версии — они общаются с `window.API`.
- **Локальный API.** В мобильной сборке `window.API` реализован в
  `api-local.js`: он разбирает те же маршруты (`/pets`, `/pets/:id/health/...`,
  `/reports/...` и т. д.) и выполняет их против `localStorage`, переиспользуя
  расчёты из `calc.js` и базу пород `breeds.js`. Благодаря этому весь экранный
  код работает без изменений и без сервера.
- **Хранение данных.** Всё лежит в `localStorage` устройства под ключом
  `hwp_mobile_v1`. Данные не покидают телефон.
- **Офлайн-оболочка.** `service-worker.js` кэширует HTML/CSS/JS, поэтому запуск
  не требует сети.

> ⚠️ Так как данные хранятся локально на устройстве, они не синхронизируются
> между телефонами. Для резервной копии используйте **Настройки → Экспорт всех
> данных** внутри приложения.

## 📦 Параметры приложения

| Параметр | Значение |
|----------|----------|
| App ID | `solutions.tracecore.helpwithapuppy` |
| Название | Help with a puppy |
| Минимальная версия Android | 5.1 (API 22) |
| Целевая версия Android | 14 (API 34) |
| Подпись APK | debug-ключ (устанавливается на любой телефон) |

> Для публикации в Google Play соберите **release**-сборку и подпишите её своим
> ключом (`./gradlew assembleRelease` + `apksigner`), затем загрузите AAB/APK в
> Play Console.
